# v0.6.0 — Phase 28: 儲存層中央集權 + 雙軌儲存

**日期**: 2026-04-30

---

## 變更摘要

### 1. 中央集權：統一 Storage 入口 (`src/storage/__init__.py`)
- 新增 `init_storage(config)` — 強制傳入 config，由高階位置決定建立哪種 Adapter
- 新增 `get_storage()` — 所有 consumer 統一入口，不關心底層是誰
- 新增 `is_initialized()` — 狀態檢查
- 解決 4 處各自初始化 SQLite 的重複問題

### 2. 雙軌儲存：地端 SQLite、雲端 DynamoDB
- 新增 `DynamoDBStorage`（`src/storage/dynamodb_adapter.py`）— 實作 `StorageInterface` 全部 5 個 method
- `StorageFactory.create()` 新增 `type=dynamodb` 分支
- `config/storage_config.json` 新增 `lambda-dev` / `lambda-prod` profiles
- **SQLite 完整保留供本地開發**

### 3. Consumer 重構（4 個檔案）
- `api_controller.py`：依 `STAGE` 環境變數自動切換（`dev/prod → DynamoDB`，其他 → SQLite）；移除 `_save_failed_request_to_llm_responses()`
- `error_logger.py`：`_init_storage()` 改 call `get_storage()`
- `utils/llm_service.py`：`_get_storage()` 改 call `get_storage()`
- `services/llm_service.py`：`_get_storage()` 改 call `get_storage()`

### 4. serverless.yml 新增 DynamoDB
- `LlmResponsesTable`（PK: trace_id, SK: created_at, GSI: organ_no, status）
- `ErrorLogsTable`（PK: trace_id, SK: created_at, GSI: error_code）
- IAM 權限：`PutItem`、`GetItem`、`Query`、`Scan`（限兩張 Table + GSI）

### 5. Bug Fix
- `test_sqlite_adapter.py` 與 `test_storage_integration.py`：`request_id` → `trace_id`（欄位已於 Phase 24 改名）

### 6. 環境切換邏輯

```
本機開發（無 STAGE）→ SQLite（data/llm_responses.db）
Lambda dev（STAGE=dev）→ DynamoDB（dev-llm-responses / dev-error-logs）
Lambda prod（STAGE=prod）→ DynamoDB（prod-llm-responses / prod-error-logs）
```
