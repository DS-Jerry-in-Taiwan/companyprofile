開發團隊的Agent分工
═══════════════════════════════════════════════

🏗️  架構師Agent (Cline - 主力)
├─ 職責: 系統架構設計、模組劃分
├─ 工具: Cline + Copilot
├─ 輸入: 需求文檔、參考架構
├─ 輸出: 目錄結構、接口定義、技術選型
└─ 階段: Phase 0 (2-3天)

💻 開發Agent (Cline - 主力)
├─ 職責: 具體模組實現
├─ 工具: Cline + Copilot (auto-complete)
├─ 輸入: 架構設計、模組規格
├─ 輸出: 可運行的代碼模組
└─ 階段: Phase 1-3 (10-12天)

🧪 測試Agent (Cline - 輔助)
├─ 職責: 單元測試、整合測試
├─ 工具: Cline + pytest
├─ 輸入: 已實現的模組
├─ 輸出: 測試代碼、覆蓋率報告
└─ 階段: 與開發並行

📚 文檔Agent (Cline - 輔助)
├─ 職責: API文檔、README、註釋
├─ 工具: Cline
├─ 輸入: 代碼模組
├─ 輸出: 完整文檔
└─ 階段: 每個模組完成後

🔧 DevOps Agent (手動 + 腳本)
├─ 職責: 環境配置、部署腳本
├─ 工具: Shell + Docker
├─ 輸入: 系統需求
├─ 輸出: 部署文檔、啟動腳本
└─ 階段: Phase 4 (2-3天)

人類開發者 (您)
├─ 職責: 決策、審查、整合、調試
├─ 工具: 大腦 + 經驗
├─ 輸入: Agent產出
├─ 輸出: 批准/拒絕、優化方向
└─ 階段: 全程
