# 1111 人力銀行 - 公司簡介 AI 生成/優化模組開發規劃 (同步 MVP 版本)

## 1. 專案背景
根據 1111 人力銀行「B.1.1 廠商基本資料」PRD (版本 0.2.4)，負責開發「公司簡介生成 API」，旨在透過 Web Search 與 LLM 技術，自動產出或優化廠商的公司簡介。
**當前階段為同步 MVP 版本，目標是打通最簡端到端流程，快速驗證核心功能。**

## 4. 應用流程 (Application Process)
1.  **觸發階段**：前端 B 端後台發送請求（包含公司名稱及 URL）。
2.  **檢索階段**：後端 **API 服務同步**觸發搜尋引擎 API，抓取 3-5 筆相關網頁資訊。
3.  **清洗階段**：**同步**提取網頁主體內容，過濾無效的 Token（廣告、Nav 等）。
4.  **推理階段**：將清洗後的資訊 + 廠商輸入 + System Prompt **同步**送入 LLM 進行推理。
5.  **後置階段**：對 AI 回傳內容進行「風險詞過濾」及「HTML 消毒」。
6.  **交付階段**：**API 服務直接返回**結構化結果供前端呈現。

## 5. 應用架構 (Application Architecture)
*   **Presentation Layer**：1111 B 端後台（React/Angular），負責渲染 AI 處理狀態與結果。
*   **Service Layer (Core API - 同步)**：
    *   **API Gateway / Controller**: 負責 API 路由、權限校驗、請求解析與錯誤處理。
    *   **Request Validator**: 驗證輸入參數。
    *   **Core Logic Dispatcher**: 根據模式分派到生成或優化模組。
    *   **Generate Brief Module**: 包含 `Web Search Service` (Serper.dev)、`Web Scraper` (BeautifulSoup)、`Text Preprocessor`、`Prompt Builder`。
    *   **Optimize Brief Module**: 包含 `Prompt Builder`。
    *   **LLM Service Wrapper**: 封裝 Gemini / OpenAI SDK。
    *   **Post-processing Module**: 包含 `HTML Sanitizer` (bleach) 和 `Content Filter` (敏感詞/競品詞)。
    *   **Response Formatter**: 格式化最終響應。
*   **Data Layer**：
    *   **PostgreSQL**：儲存最終由用戶核可的公司簡介正式資料 (MVP 階段暫不實作)。
