# 質量評估與多源決策完全指南

## 你的問題

> **"如果質量低，才啟動多源並行 — 但質量怎麼定義？"**

## 我的答案

質量評估有**三個維度**，通過加權計算得到 **0-1 的評分**，然後根據閾值決策是否多源。

---

## 一分鐘快速版

### 評分公式

```
綜合評分 = (覆蓋度 × 0.4) + (相關性 × 0.35) + (豐富度 × 0.25)
```

### 決策規則

```python
if quality_score < 0.65:
    use_multi_source = True  # 觸發多源搜尋
else:
    use_multi_source = False # 使用單源結果
```

### 評分對照

| 評分 | 含義 | 行動 |
|------|------|------|
| ≥ 0.85 | 優秀 | 直接用 |
| 0.75-0.85 | 良好 | 直接用 |
| 0.65-0.75 | 一般 | 補充 |
| 0.50-0.65 | 較差 | 必須補 |
| < 0.50 | 很差 | 重搜 |

## 評分的三個維度

### 維度 1：覆蓋度（Coverage）- 40% 權重

**問題：** 搜尋結果包含了哪些公司的關鍵信息？

**評估方法：** 檢查以下 7 項，每項存在 +0.14 分

| # | 信息項 | 檢查方式 | 例子 |
|---|--------|--------|------|
| 1 | 公司名稱 | 是否出現 | "Google Taiwan" |
| 2 | 業務描述 | 包含"業務""提供"等詞 | "提供搜尋引擎服務" |
| 3 | 產品/服務 | 包含"產品""方案"等詞 | "推出 Pixel 手機" |
| 4 | 成立時間/位置 | 包含"台灣""年份"等詞 | "2006 年成立" |
| 5 | 資質/認證 | 包含"ISO""認證"等詞 | "通過 ISO 認證" |
| 6 | 聯繫方式 | 包含"電話""官網"等詞 | "官網 google.tw" |
| 7 | 規模/營收 | 包含"員工""營收"等詞 | "員工 5000+" |

**計算示例：**
```
結果包含：公司名、業務、產品、位置、資質、聯繫方式 → 6 項
覆蓋度評分 = 6 × 0.14 = 0.84 ✓
```

---

### 維度 2：相關性（Relevance）- 35% 權重

**問題：** 搜尋結果與我的查詢有多相關？

**評估方法：** 檢查查詢詞是否出現及位置

```
查詢詞精確匹配在標題中
  → 0.95（最佳）

查詢詞精確匹配在前 100 字
  → 0.85

查詢詞 80%+ 單詞匹配
  → 0.75

查詢詞 60-80% 單詞匹配
  → 0.55

查詢詞 40-60% 單詞匹配
  → 0.35

查詢詞 < 40% 單詞匹配
  → 0.10（最差）
```

**計算示例：**
```
查詢：台灣AI新創公司
結果標題：台灣AI新創公司官網

精確匹配在標題中
相關性評分 = 0.95 ✓✓✓
```

---

### 維度 3：豐富度（Richness）- 25% 權重

**問題：** 搜尋結果有多詳細？

**評估方法：** 综合文本長度、結構化信息、結果數量

```
文本長度評分：
  < 100 字       → 0.2
  100-300 字     → 0.5
  300+ 字        → 1.0

結構化信息加分：
  有段落/列表    + 0.3
  有數字/日期    + 0.2
  有結構化格式   + 0.3

結果數量評分：
  ≥ 3 個結果     → 1.0
  2 個結果       → 0.7
  1 個結果       → 0.3
```

**計算示例：**
```
文本：250 字
結構：有段落 + 有數字 = 0.5 分
結果數：3 個

豐富度評分 = (0.5 × 0.4) + (0.5 × 0.3) + (1.0 × 0.3) = 0.65 ✓
```

## 綜合計算示例

### 例 1：大型企業（高質量）

```
查詢：Microsoft Taiwan
搜尋結果：Microsoft 官方台灣頁面 + 多篇新聞

評分：
├─ 覆蓋度 0.95 × 0.40 = 0.38
│  └─ 包含所有 7 項信息
│
├─ 相關性 0.98 × 0.35 = 0.34
│  └─ 精確匹配
│
└─ 豐富度 0.85 × 0.25 = 0.21
   └─ 文本充分 + 結構清晰 + 多結果

綜合評分 = 0.38 + 0.34 + 0.21 = 0.93 ✓✓✓
決策：使用單源（無需補充）
成本：1x
```

```python
class SearchQualityEvaluator:
    def evaluate_coverage(self, search_result):
        # 檢查 7 項信息
        score = 0
        if "公司名稱" in content:
            score += 0.14
        # ... 其他 6 項
        return score
    
    def evaluate_relevance(self, query, search_result):
        # 檢查查詢詞匹配
        # ...
        return score
    
    def evaluate_richness(self, search_result):
        # 評估文本長度、結構、結果數
        # ...
        return score
    
    def evaluate_overall(self, query, search_result):
        coverage = self.evaluate_coverage(search_result)
        relevance = self.evaluate_relevance(query, search_result)
        richness = self.evaluate_richness(search_result)
        
        overall = (coverage * 0.4 + relevance * 0.35 + richness * 0.25)
        return overall
```

### Step 2：集成到生成流程（1 天）

```python

quality_score = evaluator.evaluate_overall(query, search_result)

if quality_score < 0.65:
    # 觸發多源搜尋
    search_results = multi_source_search(queries)
else:
    # 使用單源結果
    search_results = [search_result]
```

### Step 3：記錄和監測（1 天）


- 如果用戶反饋不佳 → 降低閾值（更多多源）
- 如果成本過高 → 提高閾值（更少多源）

## 常見問題

### Q1：評分計算會不會很慢？
**A：** 不會。評分計算只需要 10-50ms，遠小於搜尋時間（1-10秒）。

### Q2：三個維度的權重（40-35-25）是怎麼來的？
**A：** 根據優化效果的優先級：
- 覆蓋度最重要（決定信息充分性）→ 40%
- 相關性次之（決定目標精準度）→ 35%
- 豐富度最後（決定詳細程度）→ 25%

可根據實際反饋調整。

### Q3：我可以自定義閾值嗎？
**A：** 完全可以。推薦開始時用 0.65，根據指標和反饋調整：
- 質量反饋不佳 → 試試 0.60
- 成本過高 → 試試 0.70

### Q4：評分會被記錄嗎？
**A：** 應該被記錄。用於：
- 監測搜尋質量趨勢
- 驗證閾值有效性
- 反饋循環優化
