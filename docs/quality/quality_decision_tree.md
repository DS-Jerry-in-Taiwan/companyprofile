# 質量評估決策樹

## 快速決策框架

### 簡化版（推薦開發初期使用）

```
開始搜尋
  │
  ▼
執行初步搜尋（快速、單源）
  │
  ▼
評估搜尋結果質量
  │
  ├─ 綜合評分 ≥ 0.85（優秀）
  │  │
  │  └─ ✓ 直接使用單源結果
  │     └─ 成本：低（1x）
  │
  ├─ 0.75-0.85（良好）
  │  │
  │  └─ ✓ 使用單源結果（可選多源）
  │     └─ 成本：低-中（1-2x）
  │     └─ 只在 30% 情況下觸發多源
  │
  ├─ 0.65-0.75（一般）
  │  │
  │  └─ ⚠ 建議多源補充
  │     └─ 成本：中（2-3x）
  │     └─ 70% 情況下觸發多源
  │
  ├─ 0.50-0.65（較差）
  │  │
  │  └─ ✗ 必須多源補充
  │     └─ 成本：高（3x）
  │     └─ 100% 觸發多源
  │
  └─ < 0.50（很差）
     │
     └─ ✗ 重新搜尋或使用備選方案
        └─ 成本：非常高（5x+）
        └─ 嘗試不同查詢或來源
```

## 詳細決策流程

### 評估維度速查表

| 維度 | 權重 | 評分方法 | 閾值 |
|------|------|--------|------|
| **覆蓋度** | 40% | 檢查 7 項關鍵信息<br/>每項 0.14 分 | ≥ 0.50 |
| **相關性** | 35% | 查詢詞匹配率<br/>位置評估 | ≥ 0.50 |
| **豐富度** | 25% | 文本長度<br/>結構化信息<br/>結果數量 | ≥ 0.30 |
| **綜合評分** | 100% | 加權平均 | ≥ 0.65 |

## 決策矩陣

### 根據評分快速判定

```python
# 最簡單的實現方式

def get_action(score: float) -> str:
    if score >= 0.85:
        return "USE"                    # 直接用
    elif score >= 0.75:
        return "USE_OPTIONALLY"         # 可用，可不用
    elif score >= 0.65:
        return "USE_WITH_SUPPLEMENT"    # 用，但補充
    elif score >= 0.50:
        return "SUPPLEMENT_REQUIRED"    # 必須補充
    else:
        return "RETRY"                  # 重新搜尋
```

## 評分計算器（一頁版）

### Step 1：評估覆蓋度 (Coverage)

檢查搜尋結果中是否包含這些信息：

| # | 信息項 | 檢查方法 | 分值 |
|---|--------|--------|------|
| 1 | 公司名稱 | 搜尋結果中出現公司名 | 0.14 |
| 2 | 業務描述 | 包含"業務""提供""主要"等詞 | 0.14 |
| 3 | 產品/服務 | 包含"產品""方案""服務"等詞 | 0.14 |
| 4 | 成立時間/位置 | 包含"台灣""成立""位於"等詞 | 0.14 |
| 5 | 資質/榮譽 | 包含"認證""ISO""獲獎"等詞 | 0.14 |
| 6 | 聯繫方式 | 包含"電話""官網""http"等 | 0.14 |
| 7 | 規模/營收 | 包含"員工""營收""融資"等詞 | 0.14 |

**覆蓋度評分 = 包含項數 × 0.14**（最高 1.0）

**例子：** 包含 5 項 → 0.70 分 ✓

---

### Step 2：評估相關性 (Relevance)

檢查搜尋結果與查詢的匹配度：

```
查詢：台灣AI新創公司
結果標題：台灣AI新創公司官網

評分規則：
✓ 精確匹配（完整公司名）        → 0.95
✓ 部分匹配（前 100 字有公司名）   → 0.85
✓ 關鍵詞匹配 ≥ 80%               → 0.75
△ 關鍵詞匹配 60-80%              → 0.55
✗ 關鍵詞匹配 40-60%              → 0.35
✗ 關鍵詞匹配 < 40%               → 0.10
```

**相關性評分 = 上述評分**

**例子：** 精確匹配 → 0.95 分 ✓✓✓

---

### Step 3：評估豐富度 (Richness)

檢查搜尋結果的詳細程度：

```
文本長度：
  < 100 字      → 0.2
  100-300 字    → 0.5
  300+ 字       → 1.0

結構化信息（加分）：
  + 有段落或列表 → +0.3
  + 有具體數字   → +0.2
  + 有結構化格式 → +0.3

搜尋結果數量：
  ≥ 3 個結果    → 1.0
  2 個結果      → 0.7
  1 個結果      → 0.3
```

**豐富度評分 = (文本長度 × 0.4) + (結構信息 × 0.3) + (結果數量 × 0.3)**

**例子：**
- 文本 250 字 → 0.5
- 有段落+數字 → 0.5
- 3 個結果 → 1.0
- 總分 = 0.5×0.4 + 0.5×0.3 + 1.0×0.3 = 0.65 ✓

---

### Step 4：計算綜合評分

```
綜合評分 = (覆蓋度 × 0.4) + (相關性 × 0.35) + (豐富度 × 0.25)
```

**例子：**
- 覆蓋度 0.70 × 0.40 = 0.28
- 相關性 0.95 × 0.35 = 0.33
- 豐富度 0.65 × 0.25 = 0.16
- **綜合評分 = 0.77** ✓

## 代碼實現（最簡版）

### 方案 A：超簡單版（1 行決策）

```python
def should_use_multi_source(quality_score: float) -> bool:
    """決定是否使用多源搜尋"""
    return quality_score < 0.65
```

### 方案 B：簡單版（分級決策）

```python
def get_action(quality_score: float) -> str:
    if quality_score >= 0.85:
        return "USE"
    elif quality_score >= 0.65:
        return "USE_WITH_OPTION"
    else:
        return "SUPPLEMENT"
```

### 方案 C：完整版（機率決策）

```python
import random

def get_action(quality_score: float) -> str:
    if quality_score >= 0.85:
        return "USE"
    
    elif quality_score >= 0.75:
        # 75-85 之間：30% 機率多源
        return "SUPPLEMENT" if random.random() < 0.3 else "USE"
    
    elif quality_score >= 0.65:
        # 65-75 之間：70% 機率多源
        return "SUPPLEMENT" if random.random() < 0.7 else "USE"
    
    else:
        return "SUPPLEMENT"  # 低於 0.65 必須多源
```

## 成本計算器

假設你每月生成 1000 份簡介：

```
閾值 0.65（推薦）：
├─ 高質量結果 (≥0.75)：500 份 × 1x = 500 單位
├─ 中等質量 (0.65-0.75)：450 份 × 2.5x = 1125 單位
├─ 低質量 (<0.65)：50 份 × 3x = 150 單位
├─ 總成本：1775 單位
└─ 平均倍數：1.78x

閾值 0.55（激進）：
├─ 平均倍數：2.2x
├─ 總成本：2200 單位
└─ 效果改善：+15%

閾值 0.75（保守）：
├─ 平均倍數：1.3x
├─ 總成本：1300 單位
└─ 效果改善：-10%
```

## 推薦方案

### 第 1 個月：使用簡單版

```python
if quality_score < 0.65:
    use_multi_source = True
else:
    use_multi_source = False
```

**理由：** 快速驗證閾值有效性

---

### 第 2 個月：收集反饋，優化閾值

根據用戶評分和指標，調整到最優值。

典型調整：
- 用戶滿意度低 → 降低閾值（更多多源）
- 成本過高 → 提高閾值（更少多源）

---

### 第 3 個月：實施機率決策

按上面"方案 C"實施，實現成本和效果的動態平衡。

## 檢查清單

實施質量評估時，確保：

- [ ] 定義了 7 項覆蓋度評分項
- [ ] 相關性評分有明確規則
- [ ] 豐富度評分算法清晰
- [ ] 綜合評分公式確定（40-35-25 權重）
- [ ] 決策閾值已定義（推薦 0.65）
- [ ] 評分結果被記錄用於監測
- [ ] 有反饋機制用於動態調整
