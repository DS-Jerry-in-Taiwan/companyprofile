# Phase 5 - Agent 角色與職責 (Agent Roles Context)

**階段**: Phase 5 - 最終測試、優化與交付

---

## 🤖 Agent 團隊分工

### 🧪 @ANALYST (測試分析 Agent - 本階段主力)
- **職責**: 執行規模化測試、評估 LLM 幻覺、產出測試報告。
- **任務**: 撰寫批次測試腳本、核對生成內容與原始資料的一致性、評比輸出內容的「吸引力」與「準確性」。
- **產出**: 完整測試評比報表、幻覺檢查分析、最終品質報告。

### 💻 @CODER (開發 Agent)
- **職責**: 根據測試反饋進行 Bug Fix 與細節優化。
- **任務**: 修復邊界測試發現的錯誤、優化 HTML 結構的穩定性、減少不必要的日誌輸出。
- **產出**: 高品質、穩定且乾淨的最終原始碼。

### 🛠️ @INFRA (DevOps Agent)
- **職責**: 建立監控工具、成本監控。
- **任務**: 統計各階段耗時、監控 API 呼叫頻率與 Token 消耗。
- **產出**: 系統效能統計表、運維監控文件。

### 🏗️ @ARCH (架構師 Agent)
- **職責**: 最終品質審核、專案打包管理。
- **任務**: 核對所有 PRD 需求的覆蓋率、完善 `README.md`、架構圖與 API 文檔。
- **產出**: 完整的專案說明文件、最終系統架構文檔。
