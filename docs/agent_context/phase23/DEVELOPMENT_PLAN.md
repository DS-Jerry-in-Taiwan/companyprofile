### 框架套用流程

```
1. 產業分類粗篩
   ↓ industry 分類
   → 排除不適用的框架
   
2. 取得資訊
   ↓ search 完成
   → 根據取得資訊決定框架複雜度
   
3. 框架組合（可隨機）
   ↓ 從剩下適合的框架中隨機選擇
   → 段落框架 + 語句框架 組合
   
4. （未來）風格樣本
   → 可加入 Few-shot 範例
```

#### 各階段輸入

| 階段 | 輸入 | 決定什麼 |
|------|------|----------|
| 1 | `industry` | 框架家族 |
| 2 | `search` 結果 | 框架複雜度 |
| 3 | 隨機或規則 | 具體框架組合 |
| 4（未來）| Few-shot | 風格樣式 |

## 問題分析

### 問題 2: 模板過於一致

**具體問題**：
> 公司段落結構大多固定（成立→理念→服務→競爭力→未來）

**當前輸出結構**：
```
[公司名稱]成立於[年份]年，資本額達[金額]，擁有[人數]名員工。[公司描述]...
```

**解決方案**：
- 文章框架庫：提供多種段落順序
- 情境庫：提供情境開頭描述
- Prompt 更新：加入多樣化指示

---

### 問題 3: 句型僵化

**具體問題**：
> 開頭固定為「XXX成立於XXX年，資本額達XXX，擁有XX名員工」開頭

**解決方案**：
- 句型庫：提供 10+ 種開頭句型
- Prompt 更新：加入開頭多樣化原則

## 📋 三個庫的內容

### 1. 文章框架庫

| 框架 | 結構 | 適用場景 |
|------|------|----------|
| 傳統結構| 成立→理念→服務→競爭力→未來 | 大眾、傳產 |
| 服務優先| 服務核心→成立→特色→未來 | 服務業 |
| 價值優先| 核心價值→成立→服務→未來 | 品牌導向 |
| 未來導向| 願景→成立→核心→服務 | 新創、科技 |
| 特色優先| 核心特色→成立→服務→未來 | 差異化公司 |
| 數據導向| 資本/規模→成立→核心→未來 | 上市公司 |

### 2. 情境庫

| 類型 | 結構 | 範例 |
|------|------|------|
| 產業情境 | 「在[產業]...[公司]...」 | 在AI快速發展的時代... |
| 市場情境 | 「面對[趨勢]...[公司]...」 | 面對數位轉型浪潮... |
| 問題情境 | 「針對[痛點]...[公司]...」 | 針對企業尋才困難... |
| 趨勢情境 | 「隨著[趨勢]...[公司]...」 | 隨著環保意識抬頭... |
| 使用者情境 | 「當你需要[服務]...[公司]...」 | 當你需要專業會計服務時... |

### 3. 句型庫

| 類型 | 結構 | 範例 |
|------|------|------|
| 服務導向 | 「專注於[產業]的[公司]，提供...」 | 常用 |
| 特色導向 | 「以[特色]聞名的[公司]...」 | 品牌 |
| 數據導向 | 「擁有[年數]年歷史的[公司]...」 | 資訊多 |
| 問句式 | 「想找[服務]嗎？[公司]是...」 | 導購 |
| 情境描述 | 「[情境]，[公司]...」 | 氛圍 |
