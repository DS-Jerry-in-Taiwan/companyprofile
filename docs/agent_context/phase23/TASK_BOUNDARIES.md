## 明確的任務邊界

### ✅ 職責範圍

| 範圍 | 說明 |
|------|------|
| **主要目標** | 解決「模板過於一致」與「句型僵化」問題 |
| **三個庫** | 文章框架庫、情境庫、句型庫 |
| **實作位置** | Prompt 模板 + 後處理模組 |

### ✅ 預期行為

| 行為 | 說明 |
|------|------|
| 開頭多樣化 | 不再固定「XXX成立於XXX年...」 |
| 段落順序變化 | 可選擇不同框架結構 |
| 情境開頭 | 可使用「在[產業]...[公司]...」等格式 |
| 隨機性 | 多次執行同一公司，開頭有變化 |

## 禁止事項

### ❌ 絕對禁止

| 項目 | 說明 |
|------|------|
| **不修改 ErrorCode** | 維持 Phase 21 的錯誤處理標準化 |
| **不改變 API 回應格式** | LLMOutput schema 維持不變 |
| **不修改現有後處理邏輯** | 不破壞 Phase 22 的 Markdown 清理 |
| **不修改 template_differentiator** | 維持現有模板差異化邏輯 |
| **不修改 word_count_validator** | 維持現有字數驗證邏輯 |

### ❌ 建議避免

| 項目 | 說明 |
|------|------|
| 不新增破壞性變更 | 確保向後相容 |
| 不增加過大模板庫 | 記憶體使用 < 10MB |
| 不過度複雜化邏輯 | 保持可維護性 |

## 決策記錄

### Q1: 模板庫放在哪裡？

| 選項 | 決定 | 理由 |
|------|------|------|
| A: 擴展 content_diversifier.py | ⏳ 待定 | 現有資源 |
| B: 新建模組 | ⏳ 待定 | 獨立模組 |
| C: Prompt | ✅ 一定要做 | 源頭解決 |

**建議**：C + (A 或 B)

### Q2: 框架選擇邏輯？

| 流程 | 決定 |
|------|------|
| 1. 產業分類粗篩 | ✅ 根據 industry 選擇框架家族 |
| 2. 取得資訊 | ✅ 根據 search 結果決定複雜度 |
| 3. 框架組合（隨機）| ✅ 可隨機選擇增加多樣性 |
| 4. 風格樣本 | ⏳ 未來考虑 |

### Q3: 三個庫要實作嗎？

| 庫 | 實作 | 說明 |
|------|------|------|
| 文章框架庫 | ⏳ 待定 | 段落順序多樣化 |
| 情境庫 | ⏳ 待定 | 開頭情境描述 |
| 句型庫 | ⏳ 待定 | 句型多樣化 |
