### 3.2 开发规范

| 項目 | 規範 |
|------|------|
| 程式碼風格 | 遵循 PEP 8，使用 ruff 檢查 |
| 命名規範 | snake_case（檔案、函式）、PascalCase（類別） |
| 單元測試 | 每個新模組需有對應的單元測試 |
| 文件註解 | 所有公開 API 需有 docstring |
| 版本控制 | 遵循 Git Flow，feature 分支開發 |

## 四、驗證流程

### 4.1 驗證時機

| 驗證點 | 時機 | 負責人 |
|--------|------|--------|
| V1：單元測試 | 每個子任務完成後 | 開發者 |
| V2：整合測試 | 每個任務完成後 | QA |
| V3：E2E 測試 | 每個里程碑完成後 | QA |
| V4：效能測試 | 可選任務完成後 | 開發者 |

### 4.2 驗證標準

- 單元測試覆蓋率 > 80%
- 整合測試通過率 = 100%
- E2E 測試通過率不降低
- 回應時間不超過 30 秒
