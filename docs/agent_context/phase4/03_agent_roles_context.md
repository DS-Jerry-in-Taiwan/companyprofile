# Phase 4 - Agent 角色與職責 (Agent Roles Context)

**階段**: Phase 4 - 安全防護與內容審核 (Risk Control)

---

## 🤖 Agent 團隊分工

### 🛠️ @INFRA (DevOps Agent)
- **職責**: 設定安全庫環境與數據存儲路徑。
- **任務**: 安裝 `bleach`、配置敏感詞清單載入路徑、配置環境變數中的過濾門檻。
- **產出**: 具備 HTML 清洗與敏感詞掃描功能的基礎環境。

### 🏗️ @ARCH (架構師 Agent)
- **職責**: 安全協議與過濾策略設計。
- **任務**: 定義過濾流程的攔截行為 (阻斷 vs 標記)、繪製安全驗證序列圖、定義 `RiskStatus` 列舉類型。
- **產出**: 安全防護邏輯文檔、審核狀態流轉模型。

### 💻 @CODER (開發 Agent)
- **職責**: 實作風控過濾核心邏輯。
- **任務**: 撰寫正則表達式掃描器、實作競品關鍵字替換、封裝 HTML 消毒模組。
- **產出**: `src/services/risk_control_service.py` 及其相關模組。

### 🧪 @ANALYST (測試分析 Agent)
- **職責**: 合規性測試、負面樣本評估。
- **任務**: 設計針對黑名單的測試案例、模擬 XSS 注入攻擊驗證安全、核對 1111 PRD 中的合規細節。
- **產出**: 風控安全驗證報告、合規檢測通過報告。
