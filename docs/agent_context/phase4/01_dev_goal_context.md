# Phase 4 - 開發目標與需求 (Dev Goal Context)

**階段**: Phase 4 - 安全防護與內容審核 (Risk Control)
**目的**: 確保 AI 生成的內容符合 1111 平台合規性，過濾高風險字眼、競品名稱並防止安全漏洞。

## 🎯 開發目標
- 整合 **高風險字眼過濾** (博弈、詐騙、虛擬貨幣等關鍵字)。
- 實作 **競品名稱屏蔽** (104, 518, yes123, LinkedIn 等競品名稱過濾)。
- 執行 **HTML 消毒 (Sanitization)**，過濾不安全標籤 (如 <script>)，防止 XSS 攻擊。
- 實作 **內容變更審核狀態機制**，對命中高風險字眼的內容標記為「待人工審核」。

## 📋 功能需求概要
1. **黑名單管理**: 建立可維護的敏感詞與競品名稱清單。
2. **過濾引擎**: 在 LLM 輸出後，存入 DB 前，執行雙重過濾 (Regex + LLM Verify)。
3. **HTML 安全化**: 僅保留 1111 編輯器支援的安全 HTML 標籤。
4. **介接 API**: 預留整合 1111 現有「高風險過濾」微服務的接口。

## ✅ 驗收標準 (Acceptance Criteria)
- 產出 `src/services/risk_control_service.py` 且通過測試案例。
- 測試包含博弈字眼的內容會被攔截或標記。
- 測試包含「104 人力銀行」的內容會被自動替換或刪除。
- 產出的 HTML 內容通過 `bleach` 等工具的安全檢查。
- 完成內容審核狀態流轉邏輯測試。
