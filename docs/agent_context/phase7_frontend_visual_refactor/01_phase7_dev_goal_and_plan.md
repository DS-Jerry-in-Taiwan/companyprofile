# Phase 7: Frontend Visual Full Refactor According to `demo_frontend.png`

## 目標 (Goal)
確保 frontend demo 介面在各種裝置、瀏覽器下，100% 還原設計稿 `demo_frontend.png` 所呈現之外觀與互動體驗。排版、配色、元件形狀、字型、按鈕、間距、品牌識別、RWD 響應能力皆須完全同步設計圖。

---

## 工項拆解

1. **設計比對分析**
   - 核對 demo_frontend.png 中所有細節：標題文字(位置/色彩/字型/大小/粗細)、按鈕、表單欄位、結果展示卡片、間距、分隔線、品牌相關視覺。
   - 列出與現行前端落差（如有）。

2. **主版面重構**
   - 頁面主版面改為垂直堆疊 (flex-column)，結果區在最上方，表單區塊在下。
   - 標題置中顯示、主色與副色需準確對應設計稿。

3. **元素調整與微調**
   - 所有輸入欄位、按鈕、提示區塊樣式、互動動畫皆依設計稿像素比對修正。
   - 結果卡片陰影、圓角、Layout 對齊設計。

4. **品牌化處理**
   - Logo、主標題、字色、所有細節，符合品牌規範。

5. **RWD 響應最佳化**
   - 於桌面、平板、手機等主流螢幕進行逐一比對與調校。

6. **驗收依據編列**
   - 撰寫明確驗收 checklist，標明畫面每一 UI element 的檢查細則。

---

## 產出標準 (Done Criteria)
- 使用任何瀏覽器或主流裝置打開 demo 頁，觀感與 demo_frontend.png 完全一致。
- 關鍵元件與排版在 RWD 模式下也無異動，互動流暢。
- 驗收清單（見下）全部通過

---

## 驗收檢查清單
- [ ] 標題/品牌區域排版、色彩、字重一致
- [ ] 結果顯示區、輸入表單等元件形狀比例、圓角、陰影吻合
- [ ] 按鈕配色含 hover/focus/disabled 狀態皆一致
- [ ] 分隔線、背景色無誤
- [ ] 所有欄位間距與區塊距離正確（像素精度）
- [ ] 手機與桌面自適應效果良好
- [ ] 再無舊有橫向 layout/side-by-side 遺留
