**更新日期**: 2026-04-16

---

## 概述

本文件定義搜尋結果的結構化格式，要求搜尋 LLM 直接返回 JSON 格式的四面向資訊，而非純文字。這樣可以使搜尋 LLM 在第一次就進行資訊整合，去除冗餘和雜訊，提升後續流程的輸入品質。

---

## 核心概念

### 四面向結構

搜尋結果被組織為四個互相補充的面向，形成完整的企業資訊圖景：

| 面向 | 英文名 | 核心目標 | 信息類型 |
|------|--------|----------|---------|
| 品牌實力與基本資料 | `foundation` | 了解企業的基本身份和實力 | 結構化數據 + 事實 |
| 技術產品與服務核心 | `core` | 理解企業的核心業務和技術 | 業務描述 + 技術亮點 |
| 職場環境與企業文化 | `vibe` | 感受企業的工作氛圍和文化 | 員工評價 + 文化特徵 |
| 近期動態與未來展望 | `future` | 掌握企業的發展方向和動態 | 新聞 + 融資 + 規劃 |

## 面向詳細定義

### 1. Foundation（品牌實力與基本資料）

**核心問題**：
- 這家企業是誰？
- 它有什麼基本資料和實力？
- 它的規模和背景如何？

**包含內容**：
- ✅ 公司名稱、成立時間、統一編號
- ✅ 公司地址、規模（員工數、年收入）
- ✅ 公司性質（上市/私有/國企等）
- ✅ 創始人、主要投資方
- ✅ 融資歷史、估值
- ✅ 獲得的主要獎項和認證
- ✅ 主營業務分類（行業、領域）

**排除內容**：
- ❌ 具體產品功能細節（應在 core）
- ❌ 員工評價和工作感受（應在 vibe）
- ❌ 最新新聞和動態（應在 future）

**最大長度**：500 字

**例子 1 - Google LLC**

```
Google LLC 成立於 1998 年，總部位於美國加州山景城。作為全球最大的搜尋引擎公司，Google 在搜尋、線上廣告、雲端計算和智能硬體等領域佔據領先地位。

公司員工超過 18 萬人，年收入超過 2000 億美元，市值達 2 兆美元以上。Google 是 Alphabet Inc. 的主要子公司，已於 2004 年上市（NASDAQ: GOOGL, GOOG）。

公司核心競爭力體現在：龐大的用戶基數（全球月活躍用戶 30 億+）、深厚的技術積累（AI/ML 領先）、完整的產品生態（Android、Chrome、YouTube 等）。曾多次獲得全球最佳雇主、創新企業等榮譽。
```

**例子 2 - Stripe**

```
Stripe 成立於 2010 年，由愛爾蘭籍兄弟檔 Patrick 和 John Collison 創辦，總部位於美國舊金山。作為全球領先的線上支付平台，Stripe 為全球數百萬家商戶提供支付解決方案。

員工約 14,000 人，估值達 950 億美元，是美國最有價值的私有軟體公司之一。Stripe 已完成多輪融資，最新一輪 (2024 年) 估值達 950 億美元。

主要業務包括：支付處理、訂閱管理、融資服務、國際支付等。Stripe 以其開發者友好的 API、全球擴展能力和創新精神獲得廣泛認可。

**例子 3 - 國內軟體公司**

```
網易科技成立於 2001 年，總部位於中國杭州。作為中國領先的互聯網科技公司，網易主要從事線上遊戲、音樂、教育等業務。

**排除內容**：
- ❌ 公司規模和背景（應在 foundation）

- ❌ 未來計劃和新聞動態（應在 future）

## 內容要求和規則

### 1. 長度限制

| 面向 | 最大字數 | 建議字數 | 說明 |
|------|----------|----------|------|
| foundation | 500 字 | 300-450 字 | 事實為主，精煉敘述 |
| core | 500 字 | 300-450 字 | 重點突出，邏輯清晰 |
| vibe | 500 字 | 300-450 字 | 主觀評價，舉例支撐 |
| future | 500 字 | 300-450 字 | 實時新聞，未來展望 |

**計算規則**：中文字符算 1 個字，英文單詞和數字算 1 個字，標點符號不計算。

### 2. 去重和優先級

**去重規則**：
- 同一面向內的重複信息，只保留最重要的版本
- 跨面向的邊界信息，優先分配到更相關的面向
  - 例：「採用 AI 技術」既涉及 core，也涉及 future，優先放在 core
  - 例：「支援遠程工作」既涉及 vibe，也涉及 foundation，優先放在 vibe

**優先級規則**（同一信息的多個版本時，優先採納）：
1. 最新信息優先（越近期越好）
2. 官方信息優先（官網 > 新聞稿 > 媒體報導）
3. 具體數字優先（「年薪 15 萬」 > 「薪酬競爭力強」）
4. 認可信息優先（獲獎、認證 > 自我宣傳）

### 3. 無法獲得信息時的處理

如果某個面向找不到相關信息，返回以下標準化文本：

```
【暫無相關資訊】該面向的相關資訊暫時無法獲取。如有需要，建議直接聯絡公司或訪問官方渠道。
```

不能留空或省略，保持結構的完整性。

### 4. 語言和風格

- **語言**：簡體中文或繁體中文，保持一致
- **風格**：專業、客觀、易讀
  - ✅ 「Google 員工超過 18 萬人」
  - ❌ 「Google 是個很大的公司」
- **時態**：現在時為主
  - ✅ 「Google 提供搜尋和廣告服務」
  - ❌ 「Google 將提供...」

### 5. 敏感信息處理

- **薪酬**：使用行業平均和範圍，而非具體個人薪資
- **個人隱私**：避免涉及個人的負面評價
- **商業機密**：不披露公司未公開的戰略和財務信息
- **政治敏感**：避免政治表態，描述事實即可

## JSON 格式驗證規則

### 有效性檢查

搜尋工具應驗證返回的 JSON 是否符合以下規則：

```python
def validate_search_result_json(json_result):
    """驗證搜尋結果 JSON 的有效性"""
    
    # 1. 必須是有效 JSON
    try:
        data = json.loads(json_result)
    except json.JSONDecodeError:
        return False, "Invalid JSON format"
    
    # 2. 必須包含四個鍵
    required_keys = {"foundation", "core", "vibe", "future"}
    if not required_keys.issubset(set(data.keys())):
        return False, f"Missing keys: {required_keys - set(data.keys())}"
    
    # 3. 所有值必須是字符串
    for key in required_keys:
        if not isinstance(data[key], str):
            return False, f"Value of '{key}' is not a string"
    
    # 4. 字數控制（500 字以內）
    for key in required_keys:
        char_count = len(data[key])
        if char_count > 500:
            return False, f"'{key}' exceeds 500 characters (got {char_count})"
    
    # 5. 不能全為空
    if all(not v.strip() for v in data.values()):
        return False, "All values are empty"
    
    return True, "Valid"
```

### Fallback 機制

如果 JSON 解析或驗證失敗，應採用以下 Fallback 策略：

1. **Parse 失敗**：嘗試尋找 JSON 部分並提取
2. **格式不完整**：填入默認的「暫無相關資訊」
3. **完全失敗**：返回原始搜尋結果，由後續流程處理

## 附錄 A：常見問題（FAQ）

### Q1: 為什麼選擇 500 字的限制？

**A**：
- 足夠詳細（比 200 字的摘要更豐富）
- 又不會太冗長（讓信息集中在核心內容）
- 四個面向共 2000 字，適合 Generate 的 token 限制
- 與行業摘要標準相符

### Q2: 如果公司很小，找不到 4 個面向的信息怎麼辦？

**A**：
- 使用「【暫無相關資訊】」填補空缺，保持結構完整
- 不要省略任何面向，這樣 Summary Node 的合併邏輯才能統一
- 後續 Generate 階段可以自動處理缺失信息

### Q3: 如何處理多語言信息？

**A**：
- 優先使用中文信息
- 如果中文信息缺失，可以翻譯英文信息
- 翻譯後的信息優先級低於原語言信息

### Q4: 是否可以加入多媒體內容（圖片、視頻等）？

**A**：
- 目前不支援，只保留文本
- 如果需要引用視頻或圖片，在文本中用 URL 連結即可
- 例：「詳見 YouTube 頻道：[URL]」

### Q5: 每個面向的欄位可以調整嗎？

**A**：
- 固定為四個面向，不調整
- 這是 Phase 16 的核心設計決策，涉及後續 Summary Node 和 Generate Node 的邏輯
- 如需調整，需要經過完整的設計評審
