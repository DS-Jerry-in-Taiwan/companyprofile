## 🔄 階段二更新：字數限制優化 (2026-04-13)

### 背景

根據用戶反饋，目前的字數限制實現存在問題：
- 強制截斷 + `...` 的方式破壞內容完整性
- 用戶體驗不佳

### 優化計劃

**負責代理人**:
- 開發代理人 B（字數檢核模組）
- 開發代理人 F/G（Prompt 層優化、整合測試）

**行動計劃**:
1. **Prompt 層字數控制強化** ✅
   - 在 prompt 中更明確地強調字數範圍
   - 加入「生成後自我檢查」指令
   - 提供字數範圍的上下限（而非只有上限）

2. **新建字數檢核模組** ✅
   - 檢查 LLM 生成的內容是否在目標字數範圍內
   - 若超出範圍，觸發重寫流程
   - 重寫時保留原有風格與意涵，只做文字縮放

3. **簡化強制截斷模組** ✅
   - 移除 `_truncate_html_content()` 和 `_truncate_plain_text()` 中的截斷邏輯
   - 改為只做輕量驗證

4. **整合到 LangGraph 流程** ✅
   - 在 `finalize_state()` 中整合字數檢核
   - 若需要重寫，呼叫 LLM 進行縮放生成

**成功指標**:
- ✅ Concise 模板：40-120 字，端到端測試 10/10 達標
- ⚠️ Standard 模板：130-230 字，端到端測試 8/10 達標 (2 超標)
- ⚠️ Detailed 模板：280-550 字，端到端測試 7/10 達標 (3 超標)
- ✅ 無突兀截斷（移除 `...` 截斷符號）
- ✅ 風格保留（重寫 prompt 設計保留風格）
- ✅ 單元測試 107/107 通過

**預估工時**: 4 天
**實際工時**: 2026-04-13 (1 天)

- `CONTENT_LENGTH_OPTIMIZATION_PLAN.md` - 完整分析報告
