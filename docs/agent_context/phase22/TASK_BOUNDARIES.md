## ❌ 禁止事項

### 絕對禁止

| 禁止項目 | 原因 |
|----------|------|
| **禁止修改 ErrorCode 枚舉** | Phase 21 已實作，不可更動 |

| **禁止修改 LLM API 呼叫參數** | 可能影響現有功能 |

| **禁止修改 `company_brief_graph.py`** | 不在此次修改範圍 |

## 決策點

### Q1: 整合到哪個檔案？

| 選項 | 優點 | 缺點 |
|------|------|------|
| **A: llm_service.py** | 最早清理，影響所有下游 | 修改核心檔案 |
| **B: post_processing.py** | 不修改核心檔案 | 較晚清理 |

**建議**: 選項 B（`post_processing.py`），風險較低

### Q2: 是否處理 `*italic*`？

| 選項 | 決定 |
|------|------|
| 處理 `*italic*` | 清理更完整，但可能影響數學表示 |
| 不處理 `*italic*` | Phase 22 不實作 |

**建議**: Phase 22 不處理 `*italic*`，專注 `**` 和 `#`

### Q3: 是否更新 Prompt 範例？

| 選項 | 決定 |
|------|------|
| 是 | Step 5 可選實作 |
| 否 | 純後處理方式 |

**建議**: Step 5 可選，取決於測試結果

## 失敗條件

如果出現以下情況，Phase 22 應停止並回滾：

| 條件 | 說明 |
|------|------|
| 單元測試失敗 > 1 個 | 10/10 必須全部通過 |
| 整合測試失敗 > 1 個 | 7/7 必須全部通過 |
| 迴歸測試失敗 | 現有功能不可被破壞 |
| API 回應格式改變 | 破壞現有介面 |
| ErrorCode 行為改變 | Phase 21 成果不可被破壞 |
